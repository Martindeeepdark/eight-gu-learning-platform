# Constitution - 项目立宪

## 项目名称
八股文可视化学习平台 (Eight-Gu Learning Platform)

## 核心使命
通过可视化、交互式的方式，帮助开发者系统化学习技术面试中的"八股文"知识，提升面试准备效率。

## 核心价值观

### 1. **用户体验优先**
- 清晰直观的知识可视化展示
- 流畅的交互体验
- 个性化的学习路径

### 2. **知识体系化**
- 将零散的八股文知识点结构化
- 建立知识点之间的关联关系
- 提供完整的学习路线图

### 3. **数据驱动**
- 追踪学习进度和掌握程度
- 智能推荐学习内容
- 可视化学习成果

### 4. **技术先进性**
- 使用现代化技术栈
- 良好的代码质量和架构
- 完善的文档和部署方案

## 成功指标

### 用户体验指标
- 页面加载时间 < 2秒
- 交互响应时间 < 200ms
- 系统可用性 > 99.9%

### 学习效果指标
- 知识点覆盖完整度 > 90%
- 学习路径清晰度
- 进度跟踪准确性

### 技术指标
- 代码覆盖率 > 70%
- API 响应时间 < 300ms
- 缓存命中率 > 80%

## 约束条件

### 技术约束
- 后端必须使用 Go + Gin 框架
- 前端必须使用 React + TypeScript + Vite
- 数据库必须使用 PostgreSQL
- 缓存必须使用 Redis
- 必须提供 Docker Compose 部署方案

### 开发约束
- 代码必须符合 Go 和 ESLint 规范
- API 必须有完整文档
- 必须包含 README 和部署说明
- 项目必须完整可运行

### 设计约束
- 遵循 RESTful API 设计原则
- 前端采用组件化开发
- 数据库设计规范化
- 安全性考虑（认证、授权、SQL注入防护等）

## 质量标准
